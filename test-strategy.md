# Test Strategy – RealWorld App (Conduit)

## 1. Introduction
This document describes how testing is approached for the RealWorld (Conduit) application. It explains what will be tested, how it will be tested, and which tools will be used. The goal is to ensure that the application works as expected from a user’s perspective and that major issues are detected early.

## 2. Testing Objectives
The main objectives of testing are:
- Make sure the core features work correctly
- Catch defects before they become serious problems
- Verify that the frontend and backend work well together
- Reduce the risk of regressions after changes
- Provide clear information about the quality of the application

## 3. Test Levels
The following test levels are covered:
- **Integration testing:** validating communication between UI and API
- **System testing:** testing complete user flows end to end
- **Regression testing:** re-testing key functionality after updates

(Unit testing is handled by developers and is not part of this strategy.)
